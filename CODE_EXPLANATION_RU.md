# Подробное объяснение кода проекта

Этот файл сделан как подробная шпаргалка по проекту.
Идея такая:
- я иду по файлам сверху вниз;
- указываю строки или небольшой диапазон строк;
- объясняю, что именно этот участок делает и зачем он нужен.

Это не "каждая строка отдельно на 2000 пунктов", потому что такой документ был бы слишком тяжёлым для чтения.
Но по смыслу здесь разобран почти весь проект: основные строки, маленькие блоки и все ключевые места.

---

## 1. `include/Exceptions.hpp`

### Строки `1-3`
- `#pragma once` не даёт подключить этот заголовок несколько раз в одну единицу компиляции.
- `#include <exception>` нужен для наследования от `std::exception`.
- `#include <string>` нужен для хранения текста ошибки.

### Строки `5-16`
- Объявляется класс `IndexOutOfRange`.
- Он нужен для всех случаев, когда индекс вышел за допустимые границы.
- Поле `message` хранит готовое текстовое описание.
- Конструктор на строках `9-12` собирает строку вида `index=..., size=...`.
- Метод `what()` на строках `13-15` переопределяет стандартный интерфейс исключения и возвращает сообщение.

### Строки `18-25`
- Объявляется `EmptyCollection`.
- Сейчас этот класс в проекте почти не используется как основной путь ошибок, потому что по условиям многие пустые случаи переведены на `IndexOutOfRange`.
- Но класс оставлен как отдельный тип ошибки, если он понадобится позже.

### Строки `27-34`
- Объявляется `ElementNotFound`.
- Это исключение используется в методе `Find`, когда элемент по условию не найден.
- То есть `Find` работает в "обычной" семантике, а `TryFind` уже работает через `Option`.

## 2. `include/IEnumerator.hpp`

### Строки `1-9`
- Это минимальный интерфейс перечислителя.
- `MoveNext()` сдвигает внутреннюю позицию на следующий элемент.
- `GetCurrent()` возвращает текущий элемент.
- `Reset()` возвращает перечислитель в состояние "до первого элемента".
- Этот интерфейс нужен, чтобы последовательность можно было обходить не только по индексам, но и в стиле перечислителя.

## 3. `include/IEnumerable.hpp`

### Строки `1-8`

- Класс `Sequence<T>` наследуется от него.

## 4. `include/Option.hpp`

### Строки `1-3`
- Заголовок защищается от повторного включения.
- `std::optional` используется как внутренняя реализация контейнера "значение есть / значения нет".
- `stdexcept` нужен для выброса ошибки при попытке взять значение из `None`.

### Строки `5-10`
- Объявляется шаблонный класс `Option<T>`.
- Внутри хранится `std::optional<T> value`.
- Конструктор `Option()` приватный, чтобы создать `Option` можно было только через `Some()` или `None()`.

### Строки `12-20`
- `None()` создаёт пустой `Option`.
- `Some(const T& val)` создаёт `Option`, в котором значение есть.
- Это делает использование класса удобным и понятным в коде.

### Строки `22-29`
- `IsSome()` проверяет, что значение существует.
- `IsNone()` проверяет обратное состояние.

### Строки `31-36`
- `GetValue()` возвращает содержимое.
- Если значения нет, выбрасывается исключение.
- Это полезно там, где мы уже знаем, что `Option` не пустой.

### Строки `38-40`
- `GetValueOrDefault()` возвращает либо реальное значение, либо запасное значение по умолчанию.

## 5. `include/DynamicArray.hpp`

### Строки `1-3`
- Подключается `Exceptions.hpp`, потому что массив выбрасывает `IndexOutOfRange`.

### Строки `4-9`
- Объявляется шаблонный класс `DynamicArray<T>`.
- Поля:
- `data` — указатель на сырой массив в куче.
- `size` — фактическое число элементов.
- `capacity` — выделенная ёмкость.

### Строки `11-24`
- `EnsureCapacity(int minCapacity)` — служебный метод.
- Если текущей ёмкости хватает, он сразу выходит.
- Иначе увеличивает `capacity` сначала до `4`, а потом удваивает, пока не хватит.
- Создаёт новый массив, копирует туда старые элементы, удаляет старый буфер и обновляет указатель.
- Это классическая логика динамического массива.

### Строки `27-27`
- Пустой конструктор создаёт полностью пустой массив: без данных, с нулевым размером и ёмкостью.

### Строки `29-37`
- Конструктор из обычного массива копирует элементы в свой собственный буфер.
- Сначала проверяет, что `count` не отрицателен.
- Потом выделяет `count` ячеек и копирует каждый элемент.

### Строки `39-47`
- Конструктор по размеру создаёт массив заданной длины.
- Все элементы инициализируются значением `T()`.
- Это удобно для дальнейших `Set(...)`.

### Строки `49-54`
- Копирующий конструктор создаёт независимую копию другого `DynamicArray`.
- Это важно для immutable-структур: копии не должны делить один и тот же буфер.

### Строки `56-58`
- Деструктор освобождает память массива.

### Строки `60-65`
- `Get(int index)` проверяет границы и возвращает элемент по индексу.

### Строки `67-72`
- `Set(int index, T value)` делает то же самое, но записывает значение.

### Строки `74-76`
- `GetSize()` возвращает логический размер массива.

### Строки `78-89`
- `Resize(int newSize)` меняет размер массива.
- Если `newSize < 0`, выбрасывается исключение.
- Если новый размер больше, вызывается `EnsureCapacity`.
- Новые элементы заполняются значением `T()`.
- Если новый размер меньше, логически хвост просто "отрезается" за счёт изменения `size`.

### Строки `91-103`
- Перегруженный `operator[]` возвращает ссылку на элемент.
- Есть две версии:
- обычная — для изменения;
- `const` — для чтения.
- В обоих случаях есть проверка границ.

## 6. `include/LinkedList.hpp`

### Строки `1-2`
- Подключаются исключения.

### Строки `4-11`
- Объявляется `LinkedList<T>`.
- Внутри есть вложенная структура `Node`.
- `Node` хранит значение и указатель `next`.
- Это односвязный список.

### Строки `13-15`
- Поля списка:
- `head` — первый узел;
- `tail` — последний узел;
- `length` — длина списка.

### Строки `17-26`
- `GetNode(int index)` — внутренний метод поиска узла.
- Проверяет границы.
- Идёт от головы `index` шагов вперёд.
- Возвращает указатель на нужный узел.

### Строки `28-38`
- `CopyFrom(const LinkedList<T>& other)` копирует содержимое другого списка.
- Сначала обнуляет текущие поля.
- Потом проходит по всем узлам другого списка и вызывает `Append`.
- Это удобно, потому что не нужно отдельно вручную поддерживать хвост и длину.

### Строки `41-41`
- Пустой конструктор создаёт пустой список.

### Строки `43-50`
- Конструктор из массива добавляет элементы один за другим в конец.
- Это превращает обычный массив в связанный список.

### Строки `52-54`
- Копирующий конструктор вызывает `CopyFrom`.

### Строки `56-63`
- Деструктор проходит по всем узлам и удаляет их по одному.

### Строки `65-70`
- `GetFirst()` возвращает значение из `head`.
- Если список пуст, выбрасывается `IndexOutOfRange`.

### Строки `72-77`
- `GetLast()` работает аналогично, но для `tail`.

### Строки `79-81`
- `Get(int index)` возвращает значение найденного узла.

### Строки `83-93`
- `GetSubList(int start, int end)` создаёт новый список из диапазона.
- Проверяет границы и корректность `start/end`.
- Копирует элементы из исходного списка в новый.

### Строки `95-97`
- `GetLength()` возвращает длину списка.

### Строки `99-108`
- `Append(T item)` добавляет новый узел в конец.
- Если список пуст, новый узел становится и головой, и хвостом.
- Иначе новый узел цепляется к старому хвосту.

### Строки `110-119`
- `Prepend(T item)` добавляет новый узел в начало.
- Если список пуст, логика такая же, как в `Append`.
- Иначе новый узел указывает на старую голову, а голова обновляется.

### Строки `121-136`
- `InsertAt(T item, int index)` вставляет элемент в позицию.
- По текущему ТЗ вставка в `index == length` запрещена, поэтому там сразу будет исключение.
- Если `index == 0`, используется `Prepend`.
- Иначе ищется предыдущий узел и между ним и текущим вставляется новый.

### Строки `138-146`
- `Concat(LinkedList<T>* list)` создаёт новый список:
- сначала копию текущего;
- затем добавляет в неё все элементы второго списка.
- Сам исходный список не меняется.

### Строки `148-154`
- Перегруженный `operator[]` даёт доступ к значению узла по индексу.
- Есть `const` и не-`const` версии.

## 7. `include/Sequence.hpp`

Это центральный файл проекта.
Здесь задаётся интерфейс `Sequence<T>` и сразу реализуются многие общие алгоритмы.

### Строки `1-8`
- Подключаются исключения, интерфейсы перечисления, `Option` и стандартные утилиты.
- `tuple` и `utility` нужны для `Zip`/`Unzip`.
- `algorithm` нужен для `std::min`.

### Строки `10-14`
- Предварительные объявления `ArraySequence` и `Sequence`.
- Они нужны, чтобы типы можно было использовать раньше полного описания.

### Строки `16-28`
- Объявляется `SequenceEnumerator<T>`.
- Это отдельный класс-перечислитель для обхода последовательности.
- Поля:
- `sequence` — последовательность, которую обходим;
- `index` — текущая позиция.
- Изначально индекс равен `-1`, то есть "до первого элемента".

### Строки `30-48`
- Объявляется абстрактный базовый класс `Sequence<T>`.
- Он наследуется от `IEnumerable<T>`.
- Методы `Get`, `GetLength`, `GetFirst`, `GetLast` — это интерфейс чтения.
- Методы `Append`, `Prepend`, `InsertAt`, `Concat`, `GetSubsequence` — интерфейс операций над последовательностью.
- `CreateEmpty()` и `CreateFromArray(...)` — это виртуальные фабрики.
- Они нужны, чтобы общие алгоритмы создавали результат правильного динамического типа.
- `GetEnumerator()` — подключает поддержку перечислителей.

### Строки `50-61`
- `Map(Func f)` применяет функцию к каждому элементу.
- Создаётся пустая последовательность того же "семейства".
- Потом для каждого элемента строится новый результат через `Append`.
- Логика с `updated != result` нужна, чтобы поддерживать и mutable-, и immutable-варианты:
- mutable вернёт тот же указатель;
- immutable вернёт новую копию.

### Строки `63-76`
- `FlatMap(Func f)` похож на `Map`, но функция возвращает уже целую последовательность.
- Каждая такая последовательность затем приклеивается через `Concat`.
- После этого промежуточная последовательность удаляется, чтобы не было утечек.

### Строки `78-85`
- `Reduce(Func f, U initial)` сворачивает последовательность в одно значение.
- Здесь используется порядок `f(currentValue, accumulator)`.
- Это соответствует формуле, которую мы держим как основную при реализации.

### Строки `87-101`
- `Where(Predicate pred)` фильтрует элементы.
- Все подходящие значения добавляются в новый результат.

### Строки `103-112`
- `Find(Predicate pred)` ищет первый элемент, удовлетворяющий условию.
- Если элемент найден, возвращается обычный `T`.
- Если нет, выбрасывается `ElementNotFound`.

### Строки `114-122`
- `Zip(Sequence<U>* other)` соединяет две последовательности в последовательность кортежей.
- Длина берётся по минимуму из двух.
- Результат — `ArraySequence<std::tuple<T, U>>`.

### Строки `124-140`
- `Unzip()` — обратная операция к `Zip`.
- Этот метод предполагает, что `T` сам является кортежем из двух элементов.
- По каждому кортежу строятся две отдельные последовательности.

### Строки `142-170`
- `Split(Predicate pred)` режет последовательность на фрагменты.
- Разделителями являются элементы, для которых предикат вернул `true`.
- Эти элементы не попадают в сами фрагменты.
- Когда встречаем разделитель, текущий фрагмент, если он не пустой, добавляется в результат.

### Строки `172-220`
- `Slice(int index, int count, Sequence<T>* replacement = nullptr)` реализует "вырезать и заменить".
- Если `index` отрицательный, он пересчитывается от конца.
- Если индекс за пределами — исключение.
- Сначала в результат копируется префикс до `index`.
- Потом, если `replacement` не пустой, вставляются его элементы.
- Потом пропускаются `count` элементов исходной последовательности.
- Затем копируется хвост.

### Строки `222-227`
- `TryGet(int index)` — безопасная версия `Get`.
- Возвращает `Option<T>`.

### Строки `229-238`
- `TryFind(Predicate pred)` — безопасная версия `Find`.
- Если элемент найден, возвращается `Some`.
- Иначе `None`.

### Строки `240-252`
- `TryFirst()` и `TryLast()` — безопасные версии доступа к первому и последнему элементу.

### Строки `254-256`
- Константный `operator[]` просто делегирует в `Get(index)`.

### Строки `258-261`
- Статический `From(...)` создаёт последовательность из обычного массива.
- Для этого используется объект-фабрика `ArraySequence<T>`.

### Строки `264-270`
- `SequenceEnumerator::MoveNext()` сдвигает текущую позицию.
- Если следующий элемент уже вне диапазона, возвращает `false`.

### Строки `273-279`
- `GetCurrent()` отдаёт текущий элемент перечислителя.
- Если перечислитель ещё не сдвинут или уже вышел за конец, выбрасывается исключение.

### Строки `281-284`
- `Reset()` возвращает индекс в стартовое состояние `-1`.

### Строки `286-289`
- `Sequence::GetEnumerator()` создаёт новый перечислитель для текущей последовательности.

## 8. `include/ArraySequence.hpp`

### Строки `1-4`
- Подключаются зависимости:
- массив;
- список;
- базовый интерфейс последовательности.

### Строки `6-17`
- Объявляется `ArraySequence<T>`.
- Поле `items` хранит внутренний `DynamicArray<T>`.
- `Clone()` создаёт копию объекта.
- `Instance()` по умолчанию возвращает `this`.
- Именно этот механизм потом переопределяется в immutable-наследнике.

### Строки `19-22`
- `AppendInternal` — низкоуровневое добавление в конец.
- Здесь уже нет логики mutable/immutable, только сама операция.

### Строки `24-32`
- `PrependInternal` создаёт новый массив на один элемент больше.
- Первый элемент заполняется новым значением.
- Старые элементы сдвигаются вправо.

### Строки `34-49`
- `InsertAtInternal` вставляет элемент в позицию.
- Проверяет индекс по текущему контракту.
- Создаёт новый буфер и копирует части массива до и после вставки.

### Строки `51-55`
- `ConcatInternal` просто многократно вызывает `AppendInternal` по элементам другой последовательности.

### Строки `57-72`
- Набор конструкторов:
- пустой;
- из массива;
- из `DynamicArray`;
- копирующий;
- из `LinkedList`;
- из указателя на `LinkedList`.
- Это покрывает все основные сценарии создания array-последовательности.

### Строки `74-76`
- Деструктор освобождает `items`.

### Строки `78-95`
- Реализация операций чтения:
- `Get`;
- `GetLength`;
- `GetFirst`;
- `GetLast`.
- `GetLast` отдельно проверяет пустой случай.

### Строки `97-103`
- Реализация фабрик `CreateEmpty` и `CreateFromArray`.
- Они нужны общим методам из `Sequence`.

### Строки `105-127`
- Реализация `Append`, `Prepend`, `InsertAt`, `Concat`.
- У всех одинаковая схема:
- взять `result = Instance()`;
- выполнить внутреннюю операцию над `result`;
- вернуть `result`.
- За счёт этого mutable-версия меняет текущий объект, а immutable-версия сначала создаёт копию.

### Строки `129-143`
- `GetSubsequence` создаёт новую последовательность из диапазона.
- Делается через общую фабрику и поэлементный `Append`.

### Строки `145-151`
- `operator[]` возвращает ссылку на элемент внутреннего массива.

## 10. Маленькие наследники mutable/immutable

Эти файлы маленькие, но очень важные.
Именно они определяют поведение "меняем объект" или "создаём копию".

### `include/Mutable/MutableArraySequence.hpp`

#### Строки `1-2`
- Подключается базовый `ArraySequence`.

#### Строки `4-13`
- `Clone()` создаёт копию mutable-объекта.
- `Instance()` возвращает `this`.
- Значит все операции применяются к текущему объекту.

#### Строки `15-29`
- Набор конструкторов и деструктор.
- Всё в основном делегируется базовому классу.

#### Строки `31-37`
- `CreateEmpty()` и `CreateFromArray()` теперь возвращают именно `MutableArraySequence`.

### `include/Immutable/ImmutableArraySequence.hpp`

#### Строки `4-13`
- `Clone()` создаёт копию.
- `Instance()` возвращает не `this`, а `Clone()`.
- Это и есть ключ immutable-логики: изменять будем не исходный объект, а его копию.

#### Строки `15-37`
- Конструкторы и фабрики аналогичны mutable-версии, но тип результата — immutable.

### `include/Mutable/MutableListSequence.hpp`

#### Строки `4-13`
- Та же идея, но для list-структуры:
- `Instance()` возвращает `this`.

#### Строки `15-35`
- Конструкторы и фабрики list-варианта.

### `include/Immutable/ImmutableListSequence.hpp`

#### Строки `4-13`
- `Instance()` возвращает копию, а не текущий объект.

#### Строки `15-35`
- Конструкторы и фабрики immutable list-варианта.

## 11. `src/main.cpp`

## 14. Что важно запомнить по архитектуре

- `DynamicArray` и `LinkedList` — это низкий уровень хранения данных.
- `Sequence` — это общий абстрактный интерфейс и место, где лежат общие алгоритмы.
- `ArraySequence` и `ListSequence` — адаптеры над двумя способами хранения.
- `Mutable*` и `Immutable*` отличаются почти только методом `Instance()`.
- `Option<T>` нужен для try-семантики.
- `IEnumerator` и `IEnumerable` нужны для обхода без индексов.
- `main.cpp` — демонстрация и ручная проверка.
- `tests.cpp` — автоматическая проверка корректности.
